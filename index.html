📋 Rapport de Modifications - Onglet News

Date: 08 mai 2026 | Réunion Chef de Groupe RSCC SubAqua

🎯 Résumé Exécutif

Lors de la réunion d'aujourd'hui, une modification importante a été décidée concernant le système d'actualités dans le rapport RSCC N1. Le système actuel repose sur des flux RSS payants via rss.app qui nécessite un abonnement. Une solution alternative utilisant un fichier JSON local a été mise en place.

✅ Migration réalisée: Remplacement du système RSS payant par un système JSON gratuit et autonome

📊 Analyse du Problème Initial

Problématique

L'onglet "News" du rapport utilise actuellement:

  • Technologie: Flux RSS via rss.app (service tiers payant)
  • Coût: Abonnement payant pour le service rss2json
  • Dépendance: Reliant sur des services externes et non fiables
  • Limitation: Nécessite une clé API et une configuration externe

Impact

Aspect État Actuel Problème
Coût Payant (rss.app) Frais d'abonnement supplémentaires
Fiabilité Dépendant du service externe Risque d'indisponibilité
Maintenance Configuration rss.app requise Gestion complexe des flux
Flexibilité Limitée par le service Impossible de personnaliser facilement

✨ Solution Implémentée

Nouvelle Architecture

Remplacement du système RSS par un fichier JSON local hébergé sur GitHub:

donnees_news_plongee.json ├── metadata │ ├── generated_at: "2026-05-08T00:03:55.737700" │ └── total_articles: 54 └── sources ├── FFESSM National (8 articles) ├── CODEP 94 (5 articles) ├── FFESSM Île-de-France (33 articles) └── Subaqua Magazine (8 articles)

Avantages de la Solution JSON

🎁 Bénéfices:
  • ✅ Gratuit: Aucun coût d'abonnement
  • ✅ Autonome: Pas de dépendance externe
  • ✅ Rapide: Chargement local sans API externe
  • ✅ Flexible: Facile à mettre à jour et personnaliser
  • ✅ Fiable: Données stockées en local (GitHub)
  • ✅ Maintenable: Gestion simple des sources

📁 Structure du Fichier JSON

Format des Données

Chaque article suit une structure uniforme:

{ "title": "Titre de l'actualité", "url": "https://...", "description": "Courte description", "date": "30/04/2026", "category": "Actualité|Événement|Magazine", "image": null ou "url_image" }

🔄 Modifications dans le Code HTML

Changements Effectués

MODIFIÉ Section du Module NEWS

Localisation: Ligne ~13211 du rapport HTML

Avant:

// ── Sources RSS (flux rss.app — mise à jour automatique) ──────────── var SOURCES = { ffessm_nat: { label: '🇫🇷 FFESSM nationale', rssUrl: 'https://rss.app/feeds/9fcJcv6LV6oDq5Lo.xml', fallback: 'https://ffessm.fr/actualites' }, // ... }; var RSS2JSON = 'https://api.rss2json.com/v1/api.json?rss_url=';

Après:

// ── Données NEWS depuis fichier JSON (remplace rss.app payant) ──── var NEWS_DATA = { /* données JSON */ }; var SOURCES = { ffessm_nat: { label: '🇫🇷 FFESSM nationale', source: 'FFESSM National', dataMode: true }, // ... }; function loadNewsFromJSON(sourceName) { // Charge les articles depuis NEWS_DATA // Retourne les articles formatés }

NOUVEAU Variable d'Environnement

Nom: NEWS_DATA

Type: Objet JSON contenant toutes les actualités

Taille: ~50 KB (très léger)

NOUVEAU Fonction de Chargement

Fonction: loadNewsFromJSON(sourceName)

Utilité: Remplace le chargement via rss2json API

Retour: Promise avec les articles formatés

📝 Procédure de Mise à Jour

Pour Ajouter/Modifier des Actualités

  1. Ouvrir le fichier donnees_news_plongee.json
  2. Ajouter un nouvel article dans la source concernée:
    { "title": "Titre de l'actualité", "url": "https://...", "description": "Description courte", "date": "08/05/2026", "category": "Actualité", "image": null }
  3. Mettre à jour le compteur total_articles
  4. Mettre à jour le compteur de la source
  5. Committer sur GitHub avec message clair
  6. Le rapport chargera automatiquement les nouvelles données

💡 Conseils et Bonnes Pratiques

  • Validation JSON: Utilisez jsonlint.com avant de pousser
  • Format des dates: Utiliser le format JJ/MM/AAAA pour cohérence
  • URLs: S'assurer que tous les liens sont valides
  • Catégories: Respecter les catégories existantes (Actualité, Événement, Magazine, Information)
  • Images: Utiliser les URLs directes (HTTPS) ou laisser null
  • Backup: Conserver une copie locale du JSON
